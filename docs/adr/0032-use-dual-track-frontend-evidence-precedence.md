# Use dual-track frontend evidence precedence

Greedy Sweeper resolves behavior and data from local `game-core` results or authoritative online state before the UI View Model and any visual artifact, while it resolves visual presentation from approved written contracts and ADRs before Visual Master `V-01`, artist-history prose, and rejected historical images. One blended ranking was rejected because generated art contains known semantic mistakes: the dual track preserves gameplay authority while still making the approved visual system reproducible.
