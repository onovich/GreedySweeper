# Split online delivery into three accepted phases

Online delivery is split into Phase 6A for workspace/shared-core/protocol foundations, Phase 6B for a playable private-room vertical slice on `workers.dev`, and Phase 6C for reconnect resilience, operations, load/security validation, progression integration, paid production, and the custom domain. A single large Phase 6 was rejected because repository migration, authoritative gameplay, browser UX, platform recovery, and production release need independent CheckAndGoal acceptance gates.
